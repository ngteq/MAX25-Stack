# MainAX25-Stack v1.0.0 scope

**Released:** `1.0.0` · Doc index: [README.md](README.md)

MAX25 is the **modem/TNC owner** — standalone-first, HyBBX-ready. HyBBX attaches to a **running** stack; it does not replace boot-wait, kernel BayCom lifecycle, or KISS PTY creation.

## Goal

1. Operator brings up **one radio path** without HyBBX (standalone).
2. Clear **HyBBX attach contract** (INI examples + lifecycle hooks).
3. **Offline CI gates** pass on Linux (`./scripts/release-check.sh`).

## Active for v1.0.0

### Devices

| Device | Status | Standalone | HyBBX |
|--------|--------|------------|-------|
| **tnc2c** | `active` | `max25-ctl start --hardware tncs --device tnc2c` | `packet_radio` |
| **baycom-ser12** | `active` | `baycom-pr-ctl start` | `baycom` |
| **soft-crdop** | `active` | `max25-ctl start --hardware soft-modems` | `crdop` |

### Daemon & client

| Component | v1 status |
|-----------|-----------|
| `max25d` | Linux supervisor, M25/1, multi-device KISS bridge |
| `max25-terminal` | Official client — text + F10 menu |
| TCP auth | Plain-text `AUTH` when `tcp_password` set |

### Operating modes

| Mode | v1 role |
|------|---------|
| `standalone` | **Default** |
| `hybbx-edge` | Documented Secondary workflow |
| `service` | Dual-radio INI templates |

## Out of scope

| Item | Target |
|------|--------|
| `pktnc2`, `baycom-par96`, `baycom-kiss` | v1.1+ |
| minicom AX.25 fork | post-v1 |
| Full *BSD port | much later — [FREEBSD-AX25.md](FREEBSD-AX25.md) |
| HyBBX vendoring | external by design |
| RF live packet in CI | manual acceptance |

./scripts/discover-plugins.sh --json

### Pre-tag checklist

- [ ] `VERSION` → `1.0.0`
- [ ] `./scripts/release-check.sh` green
- [ ] `plugins/manifest.yaml` device statuses match reality
- [ ] `share/hybbx/*.ini.example` paths valid
- [ ] One hardware smoke per active device (manual)
- [ ] Git tag `v1.0.0`

## Operator workflows

**TNC2C:**

```bash
./scripts/build.sh
./scripts/max25-ctl start --hardware tncs --device tnc2c
max25-terminal -U /run/max25/modem.sock
```

**BayCom SER12:**

```bash
./scripts/build.sh
sudo make -C stacks/baycom-pr install
sudo ./stacks/baycom-pr/scripts/baycom-pr-ctl start
```

**CRDOP:**

```bash
./scripts/build.sh
./scripts/max25-ctl start --hardware soft-modems --device soft-crdop
```

**HyBBX edge:** MAX25 prep → merge `share/hybbx/*-edge.ini.example` → start Secondary.
