# MainAX25-Stack MAX25-Stack-v1.0.0 scope

**Product:** MAX25-Stack-v1.0.0 · **Version file:** `1.0.0` · Doc index: [README.md](README.md)

MAX25 is the **modem/TNC owner** — standalone-first, HyBBX-ready. HyBBX attaches to a **running** stack; it does not replace boot-wait, kernel BayCom lifecycle, or KISS PTY creation.

## DEV-Levels roadmap (post-v1)

> DEV-Levels are **approximate orientation** (*ca.*) — see [V2.0.0-SCOPE.md](V2.0.0-SCOPE.md#dev-levels-roadmap-stack-wide).

| DEV-Level | Focus (approx.) |
|-----------|-----------------|
| **DEV-Level 1** (*ca.* current) | Modular TCP/IP Servers Service · Linux + FreeBSD compat · `max25d0` · platform extras |
| **DEV-Level 2** (*ca.*) | Main + Secondary supervision · cross-host wiring · `max25-tun` sidecar |
| **DEV-Level 3** (*ca.*) | WebSocket gateway / browser terminal · other mid-tier items (not CRDOP) |
| **DEV-Level 4** (*ca.*) | CRDOP expansion (OSS polish, G3RUH, deeper integration) |
| **Later** | AI / assistant · like-features — deferred |

v1 ships CRDOP at **minimal/native** state (build, backend, bench DSP). Major CRDOP work waits for **DEV-Level 4**; WebSocket for **DEV-Level 3**.

## Goal

1. Operator brings up **one radio path** without HyBBX (standalone).
2. Clear **HyBBX attach contract** (INI examples + lifecycle hooks).
3. **Offline CI gates** pass on Linux (`./scripts/release-check.sh`).

### MAX25-SoftModem (CRDOP) — standard

**Static project rule:** [CRDOP.md](CRDOP.md) — **CRDOP** = stack acronym for **MAX25-SoftModem** (device id `soft-crdop`) · Subproject: `$SRC/stacks/crdop/` · **GNU GPLv3**

| Topic | MAX25-Stack-v1.0.0 |
|-------|----------|
| Build | **Default ON** (`MAX25_BUILD_CRDOP=ON`) — disable only if not needed |
| Phase | **Development and test phase** — in-house development, openly documented |
| Product | In-house **MAX25-SoftModem** — 1200 baud+, half/full duplex |
| AX.25 / Packet Radio | **Acoustically fully compatible** with TNCs and BayCom modems |
| Use | Sound card IN/OUT + radio — **like a hardware modem** in max25d |

| DIY hardware | Open documentation for interface builders |

Native modem DSP is in active development; v1 ships working bench DSP (Bell 202 loopback), host spec frozen, and **installs by default**. Implementation specs: [stacks/crdop/docs/INDEX.md](../stacks/crdop/docs/INDEX.md#implementation-specs-v100).

### BayCom layout (v1 default vs dual)

| Layout | v1 role |
|--------|---------|
| **Single PC-COM** (`share/baycom/baycom-pr.pccom-ttyS0-only.ini.example`) | **Default** — most typical hosts |
| **Dual kernel-ser12** (`baycom-pr.dual.ini`, `max25d.dual-baycom.ini.example`) | **Opt-in service mode** — two UARTs, unique IRQ; not when the second UART is already a TNC |

`max25-ctl` and `install-max25.sh` ship the **single-modem** template. Dual `/etc/baycom/baycom-pr.ini` is skipped unless `--baycom-ini` or `--baycom-profile dual`. See [BAYCOM.md](BAYCOM.md).

## Operator workflows

All v1 **active** devices follow the same five steps: site config → `max25d.ini` `[devices]` → `max25-ctl start` (or `auto_start`) → `max25-terminal` with `SET DEVICE` → `CONNECT` → `SEND`. See [PLUGINS-DEVICE-MODEL.md](PLUGINS-DEVICE-MODEL.md).

**TNC2C:**

```bash
./scripts/build.sh
./scripts/max25-ctl start --hardware tncs --device tnc2c
max25-terminal -U /run/max25/modem.sock
```

**BayCom SER12 (single PC-COM):**

```bash
./scripts/build.sh
sudo cp share/baycom/baycom-pr.pccom-ttyS0-only.ini.example /etc/baycom/baycom-pr.ini
sudo baycom-pr-ctl -c /etc/baycom/baycom-pr.ini setup   # once: root, fills iobase/irq
./scripts/max25-ctl start --hardware modems --device baycom-ser12
sudo cp share/max25/max25d.ini.host.example /etc/max25/max25d.ini   # enable BayCom [devices] block
sudo max25d -c /etc/max25/max25d.ini
max25-terminal -U /run/max25/modem.sock
```

See [BAYCOM.md](BAYCOM.md).

**MAX25-SoftModem (CRDOP) — standard device:**

```bash
./scripts/build.sh
# Configure sound card + CRDOP_INI — see stacks/crdop/share/crdop.ini.example
./scripts/max25-ctl start --hardware soft-modems --device soft-crdop
sudo cp share/max25/max25d.ini.host.example /etc/max25/max25d.ini
sudo max25d -c /etc/max25/max25d.ini
max25-terminal -U /run/max25/modem.sock
```

See [stacks/crdop/docs/SOFTMODEM.md](../stacks/crdop/docs/SOFTMODEM.md).

**HyBBX host:** MAX25 prep → merge `share/hybbx/*-host.ini.example` → start Secondary.
